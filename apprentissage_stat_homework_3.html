Fully-connected neural network¶

Le Deep Learning reçoit une attention très importante depuis 2012, et même si son fondement mathématique n'est pas résolu, cette classe d'algorithmes a obtenu les meilleurs résultats dans beaucoup de domaines comme la vision ou la reconnaissance de la parole. Crées en 1998, ils ne sont utilisés que depuis récemment car ils nécéssitent une puissance de calcul élevée et des tailles de base de données importantes. Cependant, les réseaux de neurones sont aussi viables pour des problèmes classiques d'apprentissage supervisé.

Nous avons utilisé Keras avec TensorFlow pour créer un réseau à 2 couches cachées qui sont fully-connected. La sortie de l'algorithme est la probabilité que l'individu soit sain. La loss function utilisée est la logloss (ou cross-entropy). Soit $y_i$ le label de l'individu $i$ (0 pour Sain, 1 pour Malade), et soit $\widehat{p}_i$ la probabilité estimée d'appartenance à la classe 1 $\widehat{p}_i=\widehat{\mathbb{P}(Y_i=1|X)}$, la logloss est :

$$ Logloss (\widehat{y},y)=-\frac{1}{n}\sum_{i=1}^{n}y_i*\log(\widehat{p}_i)+(1-y_i)*\log(1-\widehat{p}_i)$$

Les fonctions d'activations sont RELU pour les deus couches cachées, et sigmoid (pour obtenir la probabilité) pour la sortie. L'optimiseur utilisé est Adam.

Discussion autour de l'optimalité de la solution¶

Globalement le réseau de neurones a les meilleurs résultats si l'on est indifférent des classes. Cependant, nous sommes dans un cas de détéction de maladie, donc il se peut que les deux types d'erreurs ne se paient pas au même prix en pratique.

Ici, il est fortement possible que prédire qu'un invidivu est sain alors qu'il est réellement malade soit dramatique, tandis que prédire qu'un individu est malade alors qu'il ne l'est pas vraiment soit moins grave. Ainsi, si l'on se trouve dans la situation suivante :

$$ \mbox{Coût}(\widehat{y}_i=1\ | \ y_i=0) >> \mbox{Coût}(\widehat{y}_i=0\ | \ y_i=1) $$

il faut bien évidemment adapter l'apprentissage au type de problème.

Nous allons considérer les modèles SVM et Fully-connected neural network et appliquer des poids judicieux $(w_0,w_1)$ tels que $w_0 > w_1$ et $w_0+w_1=1$ qui vont donner plus d'importance à la classe Malade. Pour SVM, le paramètre C sera différent pour chacune des deux classes. Pour la classe $i$, le paramètre $C_i$ sera $w_i*C$. Pour le réseau de neurones, pour allons directement influencer la loss function, qui sera maintenant :

$$ Logloss (\widehat{y},y)=-\frac{1}{n}\sum_{i=1}^{n}w_1*y_i*\log(\widehat{p}_i)+w_0*(1-y_i)*\log(1-\widehat{p}_i)$$

Du point de vue de l'algorithme, il est maintenant primordial de ne pas faire l'erreur qui coûte cher : prédire qu'un individu est sain alors qu'il est malade.

Maintenant, nous allons comparer les matrices de confusion avant et après l'utilisation des poids $(w_0,w_1)$ :

On peut voir que les algorithmes agissent exactement comme prévu, ils se trompent bien plus sur l'erreur qui coûte pas cher, et font beaucoup moins d'erreurs graves. Pour le réseau de neurones, cela se paye par une légère baisse de précision globale, mais peu importe : ce qui compte réellement est de ne pas faire l'erreur qui coûte cher. Les proportions d'erreurs se sont inversées : il y a maintenant 13 erreurs graves pour 84 erreurs bégignes contre 49 erreurs graves pour 13 erreurs bégignes avant. On peut moduler ce rapport en jouant avec les poids, ici nous avons volontairement mis un poids extrêmement fort pour pour pénaliser les erreurs graves. Pour SVM la situation est similaire, et il s'avère que l'on gagne en précision globale grâce à ce procédé ! Ceci peut-être dû à une meilleure généralisation de l'algorithme, qui sait désormais mieux représenter le jeu de données.

Ainsi, comme d'habitude en Data Science, il faut mettre le problème dans son contexte pour décider de l'optimalité d'une solution. Ici, il faut faire attention aux deux types d'erreurs et à leurs conséquences différentes.

def plot_confusion_matrix(cm, classes,
                          normalize=False,
                          title='Confusion matrix',
                          cmap=plt.cm.Blues):
    """
    This function prints and plots the confusion matrix.
    Normalization can be applied by setting `normalize=True`.
    """
    plt.imshow(cm, interpolation='nearest', cmap=cmap)
    plt.title(title)
    plt.colorbar()
    tick_marks = np.arange(len(classes))
    plt.xticks(tick_marks, classes, rotation=45)
    plt.yticks(tick_marks, classes)

    if normalize:
        cm = cm.astype('float') / cm.sum(axis=1)[:, np.newaxis]
        #print("Normalized confusion matrix")
    else:
        a=1
        #print('Confusion matrix, without normalization')

    #print(cm)

    thresh = cm.max() / 2.
    for i, j in itertools.product(range(cm.shape[0]), range(cm.shape[1])):
        plt.text(j, i, cm[i, j],
                 horizontalalignment="center",
                 color="white" if cm[i, j] > thresh else "black")

    plt.tight_layout()
    plt.ylabel('True label')
    plt.xlabel('Predicted label')
