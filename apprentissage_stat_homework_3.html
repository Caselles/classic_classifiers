Fully-connected neural network¶

Le Deep Learning reoit une attention trs importante depuis 2012, et mme si son fondement mathmatique n'est pas rsolu, cette classe d'algorithmes a obtenu les meilleurs rsultats dans beaucoup de domaines comme la vision ou la reconnaissance de la parole. Cres en 1998, ils ne sont utiliss que depuis rcemment car ils ncssitent une puissance de calcul leve et des tailles de base de donnes importantes. Cependant, les rseaux de neurones sont aussi viables pour des problmes classiques d'apprentissage supervis.

Nous avons utilis Keras avec TensorFlow pour crer un rseau 2 couches caches qui sont fully-connected. La sortie de l'algorithme est la probabilit que l'individu soit sain. La loss function utilise est la logloss (ou cross-entropy). Soit $y_i$ le label de l'individu $i$ (0 pour Sain, 1 pour Malade), et soit $\widehat{p}_i$ la probabilit estime d'appartenance la classe 1 $\widehat{p}_i=\widehat{\mathbb{P}(Y_i=1|X)}$, la logloss est :

$$ Logloss (\widehat{y},y)=-\frac{1}{n}\sum_{i=1}^{n}y_i*\log(\widehat{p}_i)+(1-y_i)*\log(1-\widehat{p}_i)$$

Les fonctions d'activations sont RELU pour les deus couches caches, et sigmoid (pour obtenir la probabilit) pour la sortie. L'optimiseur utilis est Adam.

Discussion autour de l'optimalité de la solution¶

Globalement le rseau de neurones a les meilleurs rsultats si l'on est indiffrent des classes. Cependant, nous sommes dans un cas de dtction de maladie, donc il se peut que les deux types d'erreurs ne se paient pas au mme prix en pratique.

Ici, il est fortement possible que prdire qu'un invidivu est sain alors qu'il est rellement malade soit dramatique, tandis que prdire qu'un individu est malade alors qu'il ne l'est pas vraiment soit moins grave. Ainsi, si l'on se trouve dans la situation suivante :

$$ \mbox{Cot}(\widehat{y}_i=1\ | \ y_i=0) >> \mbox{Cot}(\widehat{y}_i=0\ | \ y_i=1) $$

il faut bien videmment adapter l'apprentissage au type de problme.

Nous allons considrer les modles SVM et Fully-connected neural network et appliquer des poids judicieux $(w_0,w_1)$ tels que $w_0 > w_1$ et $w_0+w_1=1$ qui vont donner plus d'importance la classe Malade. Pour SVM, le paramtre C sera diffrent pour chacune des deux classes. Pour la classe $i$, le paramtre $C_i$ sera $w_i*C$. Pour le rseau de neurones, pour allons directement influencer la loss function, qui sera maintenant :

$$ Logloss (\widehat{y},y)=-\frac{1}{n}\sum_{i=1}^{n}w_1*y_i*\log(\widehat{p}_i)+w_0*(1-y_i)*\log(1-\widehat{p}_i)$$

Du point de vue de l'algorithme, il est maintenant primordial de ne pas faire l'erreur qui cote cher : prdire qu'un individu est sain alors qu'il est malade.

Maintenant, nous allons comparer les matrices de confusion avant et aprs l'utilisation des poids $(w_0,w_1)$ :

def plot_confusion_matrix(cm, classes,
                          normalize=False,
                          title='Confusion matrix',
                          cmap=plt.cm.Blues):
    """
    This function prints and plots the confusion matrix.
    Normalization can be applied by setting `normalize=True`.
    """
    plt.imshow(cm, interpolation='nearest', cmap=cmap)
    plt.title(title)
    plt.colorbar()
    tick_marks = np.arange(len(classes))
    plt.xticks(tick_marks, classes, rotation=45)
    plt.yticks(tick_marks, classes)

    if normalize:
        cm = cm.astype('float') / cm.sum(axis=1)[:, np.newaxis]
        #print("Normalized confusion matrix")
    else:
        a=1
        #print('Confusion matrix, without normalization')

    #print(cm)

    thresh = cm.max() / 2.
    for i, j in itertools.product(range(cm.shape[0]), range(cm.shape[1])):
        plt.text(j, i, cm[i, j],
                 horizontalalignment="center",
                 color="white" if cm[i, j] > thresh else "black")

    plt.tight_layout()
    plt.ylabel('True label')
    plt.xlabel('Predicted label')
